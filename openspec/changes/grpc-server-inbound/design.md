## `createServer()` factory

```ts
export interface ServerDependencies {
  readonly useCases: {
    initiateCheckout: InitiateCheckout;
    confirmCheckout: ConfirmCheckout;
    // ... one per RPC
  };
  readonly logger: Logger;
  readonly agenticJwtVerifier: AgenticJwtVerifier;  // from stripe-agentic-commerce-p1
}

export function createServer(deps: ServerDependencies): grpc.Server {
  const server = new grpc.Server({
    'grpc.max_receive_message_length': 16 * 1024 * 1024,   // webhooks can be large
    'grpc.keepalive_time_ms': 30_000,
    'grpc.keepalive_timeout_ms': 10_000,
  });

  server.addService(PaymentsCoreService, {
    initiateCheckout: withInterceptors(makeInitiateCheckoutHandler(deps)),
    confirmCheckout: withInterceptors(makeConfirmCheckoutHandler(deps)),
    // ...
  });

  // Health + reflection for dev tooling
  server.addService(HealthService, new HealthImpl());
  ReflectionService.addServiceToServer(server, PaymentsCoreService);

  return server;
}
```

## Handler shape

```ts
export function makeInitiateCheckoutHandler(deps: ServerDependencies) {
  return async (
    call: grpc.ServerUnaryCall<InitiateCheckoutRequest, InitiateCheckoutResponse>,
    callback: grpc.sendUnaryData<InitiateCheckoutResponse>,
  ): Promise<void> => {
    try {
      const input = translators.toInitiateCheckoutInput(call.request, call.metadata);
      const result = await deps.useCases.initiateCheckout.execute(input);
      callback(null, translators.fromInitiateCheckoutResult(result));
    } catch (err) {
      callback(errorMapper.toGrpc(err));
    }
  };
}
```

Handlers are pure functions of `(deps) => (call, callback) => Promise<void>`. They perform **no domain logic** — they translate, delegate, translate back.

## Error mapping

`src/adapters/inbound/grpc/error-mapper.ts` implements the table from `application-use-cases/design.md`:

```ts
export function toGrpc(err: unknown): grpc.ServiceError {
  if (err instanceof ApplicationError) {
    switch (err.code) {
      case 'INVALID_STATE': return grpcError(grpc.status.FAILED_PRECONDITION, err.message);
      case 'CURRENCY_MISMATCH': return grpcError(grpc.status.INVALID_ARGUMENT, err.message);
      case 'IDEMPOTENCY_CONFLICT': return grpcError(grpc.status.ALREADY_EXISTS, err.message);
      case 'GATEWAY_RATE_LIMITED': return grpcError(grpc.status.RESOURCE_EXHAUSTED, err.message);
      case 'GATEWAY_AUTH_FAILED': return grpcError(grpc.status.UNAUTHENTICATED, err.message);
      case 'GATEWAY_UNAVAILABLE': return grpcError(grpc.status.UNAVAILABLE, err.message);
      case 'UNKNOWN_GATEWAY': return grpcError(grpc.status.NOT_FOUND, err.message);
    }
  }
  return grpcError(grpc.status.INTERNAL, 'internal error');  // log details server-side only
}
```

Internal errors never leak stack traces to the caller. Server-side logging captures full context with the `request-id`.

## Interceptors

Unary interceptors wrap every handler:

1. **`requestIdInterceptor`** — reads `x-request-id` from metadata or generates a ULID; attaches to a `AsyncLocalStorage` context so the logger picks it up.
2. **`loggingInterceptor`** — structured log entry: `{ rpc, requestId, consumer, latencyMs, status }`. No PII; no full request body.
3. **`authInterceptor`** — reads `x-caller-id` (required, 401 otherwise) and validates; for `InitiateAgenticPayment`, also reads `authorization: Bearer <scoped-jwt>` and delegates verification to `AgenticJwtVerifier` (declared interface here, real impl lands in `stripe-agentic-commerce-p1`).

## `main.ts` (composition root)

```ts
async function main() {
  const env = loadEnv();                // zod-validated
  const logger = createLogger(env);
  const db = createPostgres(env);

  const stripeAdapter = new StripeAdapter(env, logger);            // stripe-adapter-p0
  const onvoPayAdapter = new OnvoPayAdapter(env, logger);          // onvopay-adapter-p0
  const idempotency = new PgIdempotencyStore(db);
  const bus = new InProcessEventBus();                              // out-process bus: later change
  const registry = new GatewayRegistry({ stripe: stripeAdapter, onvopay: onvoPayAdapter });

  const useCases = wireUseCases({ registry, idempotency, bus, db, logger });

  const server = createServer({ useCases, logger, agenticJwtVerifier: new NoopJwtVerifier() });

  server.bindAsync(`0.0.0.0:${env.GRPC_PORT}`, grpc.ServerCredentials.createInsecure(), (err) => {
    if (err) { logger.error({ err }, 'bind failed'); process.exit(1); }
    logger.info({ port: env.GRPC_PORT }, 'server listening');
  });

  setupGracefulShutdown(server, logger);
}
```

The composition root is the only place `src/adapters/outbound/*` and `src/application/use-cases/*` are both imported. All wiring lives here, by design.

## Graceful shutdown

On `SIGTERM`:

1. Stop accepting new RPCs (`server.tryShutdown`).
2. Allow in-flight RPCs up to 30s.
3. Force-close (`server.forceShutdown`) if the deadline expires.
4. Close DB pool, flush logger.

## Risks

- **ts-proto Long vs bigint** — ts-proto renders `int64` as `Long` (from the `long` package) by default, but we want `bigint`. Mitigation: `buf.gen.yaml` already pins `useOptionals=messages`; add `forceLong=bigint` option in `proto-contract-v1` if not already there.
- **@grpc/grpc-js keepalive tuning** — wrong values silently cut long-running webhook deliveries. Mitigation: start with the values in `createServer()`; adjust per telemetry after first real deployment.
- **Reflection in prod** — useful for dev tooling (`grpcurl`), debatable in prod. Mitigation: reflection service is enabled in v1; a future change can gate it by env.
- **Shared-process-namespace races** — K8s native sidecars are still new; teams on 1.27/1.28 fall back to plain-container sidecars with lifecycle preStop hooks. Mitigation: manifest example shows both patterns.

## Rollback

Revert. The domain and application layers remain; only the boot path goes away. Sibling sidecars (`invoice-core`, `agentic-core`) are unaffected.
